Functional Minimum Requirements

CRUD Entity 1: Categories

✓ COMPLETED
List All Categories
GET /api/categories
  • Returns all categories with pagination metadata
  • Response: { data: [...], meta: { limit, offset, total } }
✓ COMPLETED
Get Category Details
GET /api/categories/:id
  • Returns single category or 404 if not found
✓ COMPLETED
Create Category
POST /api/categories
  • Creates new category in database
  • Returns 201 with created record
✓ COMPLETED
Update Category
PUT /api/categories/:id
  • Updates existing category
  • Returns 404 if not found
✓ COMPLETED
Delete Category
DELETE /api/categories/:id
  • Removes category from database
  • Returns 204 on success, 404 if not found

CRUD Entity 2: Users

✓ COMPLETED
List All Users
GET /api/users
  • Returns all users with pagination metadata
✓ COMPLETED
Get User Details
GET /api/users/:id
✓ COMPLETED
Create User
POST /api/users
  • Fields: firstName, lastName, email
✓ COMPLETED
Update User
PUT /api/users/:id
✓ COMPLETED
Delete User
DELETE /api/users/:id
✓ COMPLETED
Pagination: limit & offset
Both /api/categories and /api/users support:
  • limit (default: 10, max: 50)
  • offset (default: 0)
  • Response includes total count in metadata
✓ COMPLETED
Search/Filter on Field Value
Both endpoints support search parameter:
  • Categories: searches on name using LIKE
  • Users: searches on firstName, lastName, and email
✓ COMPLETED
API Documentation Page
GET / (root)
  • HTML page listing all endpoints
  • Describes parameters, response formats, error codes

Validation Requirements

✓ COMPLETED
Fields Cannot Be Empty
All required fields checked:
  • Categories: name must not be empty
  • Users: firstName, lastName, email all required
✓ COMPLETED
Numeric Fields Validation
IDs and category_id must be integers:
  • Returns 400 if non-numeric ID provided
✓ COMPLETED
Names Cannot Contain Digits
Categories and Users:
  • name field (categories) cannot contain digits
  • firstName and lastName (users) cannot contain digits
  • Returns validation error if violated
